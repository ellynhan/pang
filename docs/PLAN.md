# PLAN: Phase별 개발 목표

> 각 Phase는 **실제로 실행하고 직접 확인할 수 있는 상태**로 마무리됩니다.
> 고객님께서 플레이·확인 후 피드백을 주시면 다음 Phase에 반영합니다.
> 개발 순서는 위에서 아래로 순차 진행됩니다.

---

## Phase 1 — 메인 화면

**목표**: 게임을 실행하면 타이틀 화면이 뜨고, 버튼으로 기본 메뉴를 선택할 수 있습니다.

| 기능 | 내용 |
|------|------|
| 타이틀 화면 | PANG 로고 및 배경 표시 |
| GAME START | 버튼 클릭 시 "준비 중" 안내 (게임 미구현 상태) |
| 게임 종료 | 버튼 클릭 시 탭/앱 종료 또는 종료 안내 |

**고객님 확인 포인트**
- 화면 레이아웃과 전체적인 분위기가 팡 게임답게 느껴지는지
- 버튼 위치와 크기가 누르기 편한지
- 로고와 텍스트가 잘 보이는지

---

## Phase 2 — 게임 화면 진입 및 플레이어 표시

**목표**: 1 PLAYER START를 누르면 게임 화면으로 전환되고, 플레이어 캐릭터가 바닥에 표시됩니다.

| 기능 | 내용 |
|------|------|
| 화면 전환 | 타이틀 → 게임 화면 전환 |
| 게임 배경 | 스테이지 배경 화면 렌더링 |
| 플레이어 렌더링 | 캐릭터가 화면 하단 중앙에 표시 |
| HUD 기초 | 라이프·점수 표시 영역 (초기값 고정) |

**고객님 확인 포인트**
- 타이틀에서 게임 화면으로 전환이 자연스러운지
- 캐릭터의 크기와 위치가 적절한지
- 배경과 캐릭터가 어울리는지

---

## Phase 3 — 플레이어 이동

**목표**: 키보드로 캐릭터를 좌우로 움직일 수 있습니다.

| 기능 | 내용 |
|------|------|
| 좌우 이동 | ←/→ 키로 플레이어 이동 |
| 벽 충돌 | 화면 좌우 끝에서 이동 차단 |
| 이동 속도 | 적절한 기본 이동 속도 적용 |

**고객님 확인 포인트**
- 이동 속도가 너무 빠르거나 느리지 않은지
- 벽에 닿았을 때 자연스럽게 멈추는지
- 키 입력 반응이 즉각적으로 느껴지는지

---

## Phase 4 — 작살 발사

**목표**: 버튼을 누르면 위 방향으로 작살을 발사할 수 있습니다.

| 기능 | 내용 |
|------|------|
| 작살 발사 | 스페이스바로 위 방향 발사 |
| 천장 고정 | 작살이 천장에 닿으면 고정 후 일정 시간 뒤 소멸 |
| 발사 제한 | 작살이 살아있는 동안 재발사 불가 |

**고객님 확인 포인트**
- 작살 발사 타이밍과 속도가 자연스러운지
- 천장에 닿은 후 사라지는 타이밍이 적절한지
- 연속 발사가 막히는 것이 불편하지 않은지

---

## Phase 5 — 버블 등장 및 이동

**목표**: 버블이 화면에 나타나 물리 법칙에 따라 튀어 다닙니다. (아직 분열·충돌 없음)

| 기능 | 내용 |
|------|------|
| 버블 렌더링 | 대형 버블 2개 화면에 표시 |
| 버블 이동 | 좌우 이동 + 바닥·벽 반사 |
| 크기별 물리 | 클수록 높이 튀고 느림, 작을수록 낮고 빠름 |

**고객님 확인 포인트**
- 버블이 튀는 느낌이 자연스러운지 (탄성감)
- 벽과 바닥 반사 방향이 자연스러운지
- 버블 크기가 시각적으로 명확히 구분되는지

---

## Phase 6 — 버블 분열

**목표**: 작살이 버블에 맞으면 2개의 작은 버블로 분열하고, 극소 버블은 소멸합니다.

| 기능 | 내용 |
|------|------|
| 버블 분열 | 작살 충돌 시 한 단계 작은 버블 2개 생성 |
| 버블 소멸 | 극소 버블 충돌 시 완전 제거 |
| 분열 방향 | 분열 시 두 버블이 좌·우로 퍼짐 |

**고객님 확인 포인트**
- 버블을 맞혔을 때 분열되는 느낌이 시원한지
- 분열 후 두 버블의 이동 방향이 자연스러운지
- 연쇄 분열(대→중→소→극소)이 끊김 없이 이어지는지

---

## Phase 7 — 충돌 판정 및 라이프 시스템

**목표**: 버블에 닿으면 미스 처리되고, 라이프가 줄어듭니다.

| 기능 | 내용 |
|------|------|
| 충돌 감지 | 플레이어와 버블 접촉 감지 |
| 미스 처리 | 접촉 시 라이프 1 감소 + 무적 시간 부여 |
| HUD 업데이트 | 라이프 수 실시간 반영 |
| 스테이지 재시작 | 미스 후 같은 스테이지 처음부터 재시작 |

**고객님 확인 포인트**
- 충돌 판정 범위가 너무 넓거나 좁지 않은지
- 미스 후 재시작까지 흐름이 어색하지 않은지
- 무적 시간 동안 시각적으로 구분이 되는지 (깜빡임 등)

---

## Phase 8 — 스테이지 클리어 및 게임 오버

**목표**: 모든 버블 제거 시 클리어되고, 라이프 소진 시 게임 오버 화면이 뜹니다.

| 기능 | 내용 |
|------|------|
| 스테이지 클리어 | 버블 전체 소멸 시 클리어 판정 |
| 클리어 연출 | 클리어 메시지 표시 후 다음 스테이지 이동 |
| 게임 오버 | 라이프 소진 시 GAME OVER 화면 표시 |
| 재시작 | 게임 오버 화면에서 타이틀로 복귀 |

**고객님 확인 포인트**
- 클리어 판정이 즉각적으로 느껴지는지
- 게임 오버 화면에서 타이틀로 돌아가는 흐름이 자연스러운지
- 클리어/게임오버 화면이 너무 길거나 짧지 않은지

---

## Phase 9 — 점수 시스템

**목표**: 버블 제거 시 점수가 쌓이고, 화면에 실시간으로 반영됩니다.

| 기능 | 내용 |
|------|------|
| 버블별 점수 | 크기가 클수록 높은 점수 |
| 실시간 점수 표시 | HUD 점수 실시간 업데이트 |
| 최고 점수 | 세션 내 최고 점수 타이틀 화면에 표시 |

**고객님 확인 포인트**
- 버블을 터뜨릴 때 점수가 오르는 피드백이 느껴지는지
- 크기별 점수 차이가 납득되는지
- 최고 점수가 타이틀에서 잘 보이는지

---

## Phase 10 — Mission 1 Stage 구성

**목표**: Mission 1의 3개 스테이지가 설계된 버블 배치와 난이도로 구성됩니다.

| 기능 | 내용 |
|------|------|
| Stage 1 | 대형 버블 2개, 평지 구성 |
| Stage 2 | 대형 1 + 중형 2, 발판 1~2개 추가 |
| Stage 3 | 중형 2 + 소형 2, 발판 복수·높낮이 차이 |
| 스테이지 순서 | Stage 1 → 2 → 3 순서로 자동 진행 |

**고객님 확인 포인트**
- Stage 1 → 2 → 3으로 갈수록 난이도가 체감되는지
- 각 스테이지의 버블 배치가 지루하거나 너무 어렵지 않은지
- 발판 위치가 전략적 이동에 도움이 되는지

---

## Phase 11 — 아이템 시스템

**목표**: 버블 제거 시 아이템이 드롭되고, 먹으면 효과가 적용됩니다.

| 기능 | 내용 |
|------|------|
| 아이템 드롭 | 버블 소멸 시 확률적으로 아이템 등장 |
| 더블 와이어 | 작살 2개 동시 발사 |
| 시계 | 모든 버블 일시 정지 |
| 다이너마이트 | 화면 내 모든 버블 즉시 소멸 |
| 방어막 | 버블 접촉 1회 무적 |
| 1UP | 라이프 1 추가 |

**고객님 확인 포인트**
- 아이템이 등장할 때 눈에 잘 띄는지
- 각 아이템 효과가 플레이에 직접적인 도움이 되는지
- 아이템 드롭 빈도가 너무 잦거나 드물지 않은지

---
